A. Création du MCD (Modèle Conceptuel de Données)

  1. Le MCD (Modèle Conceptuel de Données)
    est une représentation des données nécessaires à une application sans se soucier des aspects techniques de la base de données. Il sert à comprendre et définir les données qui doivent être stockées ainsi que les relations entre ces données. C'est une étape de conception importante avant de passer à la création des tables dans la base de données.

  2. Objectifs du MCD :
    • Identifier les entités principales (objets ou concepts) de l'application.
    • Définir les relations entre ces entités.
    • Indiquer les attributs (données spécifiques) de chaque entité.
    • Identifier les cardinalités des relations (par exemple, un utilisateur peut avoir plusieurs films en favoris, mais chaque film peut avoir plusieurs utilisateurs qui le considèrent comme favori).

  3. Étapes pour créer un MCD :
    1. Identification des entités : Les entités sont les objets que nous souhaitons gérer dans notre base de données. Dans le cas d'un projet "Blog-Cinéma", les entités principales peuvent être UTILISATEURS, FILMS, CATEGORIES, LISTES, etc.
    2. Définition des relations : Les relations entre les entités représentent la manière dont les entités interagissent entre elles. Par exemple, un utilisateur peut avoir plusieurs films dans ses favoris, donc il y a une relation UTILISATEUR-FAVORIS-FILM.
    3. Définition des attributs : Chaque entité aura des attributs qui décrivent ses caractéristiques. Par exemple, l'entité FILMS peut avoir des attributs comme titre, date_sortie, synopsis, etc.
    4. Identification des cardinalités : La cardinalité décrit le nombre d’occurrences d’une entité qui peuvent être associées à une autre. Par exemple, un UTILISATEUR peut avoir plusieurs FILMS dans ses FAVORIS, mais chaque FILM peut être dans plusieurs LISTES. Une relation UN à N.

  4. Exemple simplifié de MCD :
    Voici un exemple de MCD simplifié pour un projet de Blog-Cinéma :
    • UTILISATEURS : id, nom, email, mot_de_passe.
    • FILMS : id, titre, synopsis, date_sortie.
    • FAVORIS : id, utilisateur_id, film_id.
    • LISTES : id, utilisateur_id, nom, type.

    Les relations :
    • Un utilisateur peut avoir plusieurs films dans ses favoris.
    • Un utilisateur peut avoir plusieurs listes de films.
    • Un film peut appartenir à plusieurs listes.

B. Création du MLD (Modèle Logique de Données)

  1. Le MLD (Modèle Logique de Données)
    est une version plus détaillée et technique du MCD. Tandis que le MCD se concentre sur les objets métiers, le MLD inclut les informations nécessaires pour implémenter ces objets dans une base de données relationnelle.

  2. Objectifs du MLD :
    • Traduire le MCD en un modèle plus proche de la réalité d'une base de données relationnelle.
    • Définir la structure des tables de la base de données, y compris les types de données et les contraintes.
    • Ajouter les clés primaires, clés étrangères et relations entre les tables.


  3. Étapes pour créer un MLD :
    1. Définir les tables : À partir des entités définies dans le MCD, on crée des tables. Par exemple, l’entité UTILISATEUR devient une table UTILISATEURS.
    2. Définir les types de données : Chaque attribut d'une entité dans le MCD doit avoir un type de données correspondant dans le MLD. Par exemple, email pourrait être de type VARCHAR(255).
    3. Définir les clés primaires : Chaque table aura une clé primaire (généralement un identifiant unique). Par exemple, l'attribut id dans la table UTILISATEURS serait la clé primaire.
    4. Définir les clés étrangères : Les relations entre les entités du MCD sont représentées par des clés étrangères dans le MLD. Par exemple, la relation entre UTILISATEURS et FAVORIS pourrait être représentée par une clé étrangère utilisateur_id dans la table FAVORIS.
    5. Définir les contraintes : Certaines contraintes doivent être ajoutées, comme NOT NULL, UNIQUE, FOREIGN KEY, etc., pour garantir l’intégrité des données.


  4. Exemple de MLD basé sur le MCD : 
                        Voici un exemple de MLD pour notre projet Blog-Cinéma :

Table UTILISATEURS :

id INT (clé primaire)
nom VARCHAR(100) NOT NULL
prenom VARCHAR(100) NOT NULL
email VARCHAR(255) UNIQUE NOT NULL
mot_de_passe VARCHAR(255) NOT NULL
date_inscription DATE NOT NULL
Table FILMS :

id INT (clé primaire)
titre VARCHAR(255) NOT NULL
synopsis TEXT
date_sortie DATE
image VARCHAR(255)
notation FLOAT
critique TEXT
Table FAVORIS :

id INT (clé primaire)
utilisateur_id INT (clé étrangère vers UTILISATEURS)
film_id INT (clé étrangère vers FILMS)
liste_numero TINYINT NOT NULL

C. Planification des tables et relations

La planification des tables et relations consiste à traduire le MCD et le MLD en une structure relationnelle et à réfléchir à la manière dont les tables vont interagir entre elles. Cela implique :

  • Définir les tables et les relations : Après avoir créé le MCD et le MLD, vous devez vous assurer que chaque relation est bien définie et que les clés primaires et étrangères sont correctement utilisées pour établir ces relations.
  • Déterminer les cardinalités des relations : Chaque relation a une cardinalité, telle que un à un, un à plusieurs, ou plusieurs à plusieurs, et cela doit être pris en compte lors de la création des clés étrangères.
  • Optimiser les performances : Parfois, les relations peuvent être ajustées pour améliorer les performances des requêtes (en fonction des besoins du projet).

Exemple de relations dans les tables :

  • La table FAVORIS contient deux clés étrangères, une vers UTILISATEURS et une vers FILMS, ce qui établit une relation plusieurs à plusieurs entre les utilisateurs et les films dans les favoris.
  • La table FILMS_CATEGORIES est une table de jonction entre FILMS et CATEGORIES, et elle contient des clés étrangères pour chaque entité.

Conclusion

Le MCD et le MLD sont des outils essentiels pour la conception d'une base de données. Le MCD permet de comprendre les données et leurs relations à un niveau abstrait, tandis que le MLD traduit cette structure en un modèle plus technique, prêt à être implémenté dans une base de données relationnelle. La planification des tables et des relations est la dernière étape avant de commencer à implémenter la base de données et garantir que les données peuvent être manipulées efficacement et en toute sécurité.
