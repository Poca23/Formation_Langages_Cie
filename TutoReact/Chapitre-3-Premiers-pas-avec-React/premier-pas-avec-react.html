Installer son environnement de développement

Il faut choisir et installer un éditeur de code pour écrire du code React. Il est possible d'utiliser Visual Studio Code (gratuit et développé par Microsoft), Sublime Text, Atom, WebStorm, etc. Il faudra que cet éditeur de code supporte le TypeScript en s'occupant de la coloration synthaxique et de l'affichage d'erreurs propres à ce langage.

Rendez vous sur la doc du site officiel de TypeScript : TypeScript

Démarrer un projet React

Il est possible de démarrer un projet React en utilisant la commande suivante :

npx create-react-app mon-projet

Cependant, nous poartirons d'un dossier vide pour pouvoir voir le processus de création d'une nouvelle application React en partantv de zéro.

Il sera nécessaire de rajouter quelques fichiers dans notre dossier racine vide de notre projet intitulé : "REACT-POKEMONS-APP"

Commençons par la base. Nous partirons d'un dossier vide jusqu'à obtenir une application finale prête pour la production.

Le fichier package.json

On ajoutera le fichier package.json à la racine de notre projet.

Le fichier package.json est un fichier de configuration pour notre projet. Il contient des informations sur notre projet, les dépendances utilisées, les scripts disponibles, etc.

Le contenu de notre fichier package.json peut se découper en 3 parties :


Les dépendences de notre projet : 

                     "dependencies": {
                    "@types/node": "12.11.1",
                    "@types/react": "16.9.9",
                    "@types/react-dom": "16.9.2",
                    "@types/react-router-dom": "^5.1.2",
                    "react": "^17.0.0",
                    "react-dom": "^17.0.0",
                    "react-router-dom": "^5.1.2",
                    "react-scripts": "^4.0.0",
                    "typescript": "3.6.4"
                  },

Les scripts de notre projet : 

                     "scripts": {
                        "start": "react-scripts start",
                        "build": "react-scripts build"
                      },

"start" permet de mettre en place un petit environnement de développement React pour pouvoir tester notre projet react dans un navigateur.

"build" nous permet de créer le livrable de notre application une fois le developpement fini.



Vers la fin du fichier on retrouve un élément "eslintConfig" qui permettra d'analyser la syntaxe du code écrit avec es6.

                    "eslintConfig": {
                        "extends": "react-app"
                        },


L'élément "browserslist" permet de configurer le comportement de certains scripts en fonction des serveurs sur lesquels on souhaite faire marcher l'application.

Le fichier tsconfig.json

Ce fichier est un fichier de configuration pour TypeScript. Il permet de configurer le comportement de TypeScript dans notre projet. Il doit être aussi placé à la racine du projet.

Tout d'abord, nous allons le créer :

                    
                        {
                            "compilerOptions": {
                              "target": "es5",
                              "lib": [
                                "dom",
                                "dom.iterable",
                                "esnext"
                              ],
                              "allowJs": true,
                              "skipLibCheck": true,
                              "esModuleInterop": true,
                              "allowSyntheticDefaultImports": true,
                              "strict": true,
                              "forceConsistentCasingInFileNames": true,
                              "module": "esnext",
                              "moduleResolution": "node",
                              "resolveJsonModule": true,
                              "isolatedModules": true,
                              "noEmit": true,
                              "jsx": "react",
                              "noFallthroughCasesInSwitch": true
                            },
                            "include": [
                              "src"
                            ]
                          }

TSX est une syntaxe qui permet d'écrire du HTML dans du JavaScript. Elle est utilisée par React pour décrire l'interface utilisateur. 

TSX = TypeScript + JSX

JSX est une extension de la syntaxe JavaScript qui permet d'écrire du HTML directement dans le code JavaScript. Elle est utilisée par React pour décrire l'interface utilisateur de manière déclarative. 

                    
                        "include": ["src"]
                    }

L'option "include" indique au compilateur de TypeScript ou se trouve les fichiers que nous voulons compiler vers JavaScript.Nous indiquons par la même que nous placerons tous nos fichiers sources dans un dossier source "src" à TypeScript.

Installer les dépendances

Nous allons installer les bibliothèques que nous avons déclarées dans notre fichier package.json. Pour cela, nous allons utiliser la commande suivante à lancer à la racine de notre projet dans le terminal :

                    npm install

Cette commande va créer un dossier à la racine de notre projet nommé "node_modules". Il contient toutes les dépendances dont nous aurons besoin pour faire fonctionner notre projet.

Ajouter notre premier composant

Nous allons créer notre premier composant React. Mais avant cela, nous allons organiser un peu notre code en mettant nos fichier dans un dossier src, un dossier source, à la racine de notre projet.

A l'intérieur de ce dossier, créons un nouveau fichier nommé "App.tsx" qui contiendra le code de notre premier composant.

                
                    import React from 'react';
  
                    const App: React.FC = () => {
                     const name: String = 'React';
                        
                     return (
                      
Hello, {name} !

                     )
                    }
                      
                    export default App;

Ce fichier est composé de quatres parties différentes :

  • On importe React depuis le module React dont on va avoir besoin pour déclarer notre premier composant : 

                                
                                import React from 'react';

  • On s'aperçoit à la ligne 3 qu'on a typé notre composant App avec le type :

                            
                            App: React.FC

    Qu'est-ce que FC signifie ?
    FC signifie Function Component. C'est un type générique qui permet de typer les composants fonctionnels en React. IL permet d'écrire un composant sous la forme d'une focntion.

    Il y a écrit le mot const au début de la ligne 3. C'est une constante qui contient une fonction. Une fonction fléchée.
    Pourquoi passer par une constante ?
    Ça nous permet d'utiliser une fonction fléchée plutôt qu'une fonction traditionnelle.

  • On déclare en troisième partie le code de notre composant. On déclare une variable à la ligne 4 de type String qui contient la valeur React.


    Ensuite, on souhaite afficher cette valeur dans notre DOM virtuel, pour cela, on utilise la fonction return à la ligne 6. Et entre parenthèse nous écrivons l'expression JSX suivante :

                                    
                                    
    Hello, {name} !

    Comme on peut le voir, la synthaxe avec les accollades permet d'utiliser JavaScript dans JSX qui va remplacer name par la valeur "React".


    Lorsque nous aurons fini de mettre en place notre projet, c'est donc la première chose que nous verrons à l'écran.

  • La dernière ligne du fichier :

                            
                            export default App;

    Elle permet d'exporter notre composant afin qu'il soit utilisable dans le reste de notre application.

Petit tips ! Plutôt que d'écrire :

                
                    App: React.FC

Nous pouvons importer directement depuis le paquet react le type :

                
                    import React, {FunctionComponent} from 'react';

Ensuite nous pouvons utiliser cette nouvelle importation à la place de React.FC :

                
                    const App: FunctionComponent = () => {...}
