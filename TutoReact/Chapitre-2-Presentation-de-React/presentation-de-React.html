Chapitre 2 - Présentation de React

Sommaire

C'est quoi React

React est une bibliothèque JavaScript développée par Facebook.

C'est un framework : Un ensemble de composants et d'outils qui permettent de développer plus rapidement des applications web.

Elle permet de créer des interfaces utilisateur interactives et dynamiques. React est basé sur un système de composants réutilisables.

React permet de développer des applications web de manière robuste et efficace.

Différence entre application web et site web

Un site web est une collection de pages web reliées entre elles par des liens hypertextes.
Un site web est généralement statique, c'est-à-dire que le contenu des pages est fixe et ne change pas.
Les sites web sont utilisés pour présenter des informations, des produits ou des services. Ils sont généralement conçus pour être consultés par un grand nombre de personnes.



Une application web est un logiciel qui s'exécute dans un navigateur web et ne contient qu'une simple page html.
Elle permet d'effectuer des tâches spécifiques comme la gestion de données, la communication en temps réel, etc. Une application web est généralement dynamique, c'est-à-dire que le contenu des pages peut changer en fonction des actions de l'utilisateur.
Les applications web sont utilisées pour des tâches plus complexes que les sites web. Elles sont généralement conçues pour un usage spécifique.
Le serveur ne renvoie pas une nouvelle page à chaque fois que l'utilisateur effectue une action, mais met à jour la page existante en fonction des actions de l'utilisateur. Cela permet de créer des applications web plus rapides et plus réactives. C'est ce qu'on appelle le développement d'applications web côté client. Puis le JavaScript prend le relais pour mettre à jour la navigation et les contenus de la page.

On a une architecture SPA (Single Page Application). C'est une application web qui tient sur une seule page web. Elle ne nécessite pas de rechargement de page lors de la navigation. Cela permet d'avoir une expérience utilisateur plus fluide et plus rapide.

Pourquoi utiliser React ?

React est une bibliothèque JavaScript très populaire pour le développement d'applications web.
Voici quelques raisons pour lesquelles vous devriez utiliser React :

  • Composants réutilisables : React permet de créer des composants réutilisables, ce qui facilite la maintenance et la mise à jour du code.
  • Performance : Grâce au Virtual DOM, React optimise les mises à jour de l'interface utilisateur, ce qui améliore les performances.
  • Écosystème riche : React dispose d'un écosystème riche avec de nombreuses bibliothèques et outils qui facilitent le développement.
  • Communauté active : React bénéficie d'une grande communauté de développeurs, ce qui signifie qu'il y a beaucoup de ressources et de support disponibles.
  • Apprentissage et adoption : React est relativement facile à apprendre et à adopter, surtout pour les développeurs ayant des connaissances en JavaScript.

La philosophie de React

React est basé sur quelques principes fondamentaux qui guident la conception et le développement d'applications web :

  • Composants : React est basé sur un système de composants réutilisables. Les composants sont des éléments d'interface utilisateur qui peuvent être assemblés pour créer des interfaces complexes.
  • Virtual DOM : React utilise un Virtual DOM pour optimiser les mises à jour de l'interface utilisateur. Le Virtual DOM est une représentation virtuelle de l'interface utilisateur qui est mise à jour de manière efficace.
  • Unidirectionnel : React utilise un flux de données unidirectionnel pour gérer les données et les mises à jour de l'interface utilisateur. Cela permet de maintenir un état cohérent de l'application.
  • JavaScript : React est basé sur JavaScript, ce qui signifie que vous pouvez utiliser toutes les fonctionnalités du langage pour développer des applications web.

Un composant React est composé de HTML, éventuellement de CSS et d'une fonction JavaScript. Les composants React reposent sur le standard des Web Components/composants web. Il a été pensé pour diviser la page web en fonction de leurs rôles ; barre de navigation, formulaires, etc.. C'est une partie de l'application qui fonctinne de façon autonome.
