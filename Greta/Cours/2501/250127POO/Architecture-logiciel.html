Architecture logiciel

Introduction

Une architecture logicielle est une représentation des éléments d'un système logiciel, des relations entre ces éléments et des propriétés de ces éléments. L'architecture logicielle est distincte de l'architecture matérielle mais influence fortement cette dernière. Elle est également distincte du design logiciel qui est la conception de la structure interne d'un logiciel.

Principes

Les principes de l'architecture logicielle sont les suivants :

  • Modularité : découper le logiciel en modules
  • Abstraction : cacher les détails d'implémentation
  • Encapsulation : regrouper les données et les méthodes qui les manipulent
  • Héritage : réutiliser le code existant
  • Polymorphisme : utiliser des objets de manière générique

Modèles

Il existe plusieurs modèles d'architecture logicielle :

  • Modèle en couches
  • Modèle en étoile
  • Modèle en pipe and filter
  • Modèle en client-serveur
  • Modèle en microservices

Design patterns

Les design patterns sont des solutions récurrentes à des problèmes récurrents. Il existe plusieurs types de design patterns :

  • Création : factory, builder, prototype, singleton
  • Structure : adapter, bridge, composite, decorator, facade, flyweight, proxy
  • Comportement : chain of responsibility, command, interpreter, iterator, mediator, memento, observer, state, strategy, template method, visitor

Schéma de l'architecture logicielle

Conclusion

L'architecture logicielle est un élément clé de la réussite d'un projet logiciel. Elle permet de découper le logiciel en modules, de cacher les détails d'implémentation, de regrouper les données et les méthodes qui les manipulent, de réutiliser le code existant et d'utiliser des objets de manière générique. Il existe plusieurs modèles d'architecture logicielle et plusieurs design patterns qui permettent de résoudre des problèmes récurrents.
