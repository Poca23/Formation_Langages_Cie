Chapitre 10 - Introduction à Laravel

Présentation du framework Laravel

Laravel est un framework PHP open-source, puissant et élégant, qui facilite le développement d'applications web robustes. Il propose une syntaxe expressive, une grande facilité d'utilisation, et de nombreux outils intégrés pour améliorer l'efficacité du développement.

Laravel est largement utilisé pour sa capacité à simplifier des tâches courantes telles que la gestion de bases de données, l'authentification, les routages et bien plus encore. Il repose sur le patron de conception MVC (Model-View-Controller), ce qui facilite l'organisation et la maintenance du code.

Installation de Laravel

Pour commencer à utiliser Laravel, vous devez d'abord installer Composer, un gestionnaire de dépendances PHP qui facilite l'installation de Laravel et d'autres bibliothèques nécessaires.

1. Installation de Composer

Si vous n'avez pas encore installé Composer, vous pouvez le faire en suivant ces étapes :

  • Sur Linux/macOS : ouvrez votre terminal et tapez la commande suivante :
curl -sS https://getcomposer.org/installer | php
  • Sur Windows, vous pouvez télécharger Composer depuis le site officiel.

2. Création d'un projet Laravel

Une fois Composer installé, vous pouvez créer un projet Laravel en exécutant la commande suivante : 

composer create-project --prefer-dist laravel/laravel monProjetLaravel

Cela créera un dossier monProjetLaravel contenant une nouvelle application Laravel prête à l'emploi.

3. Lancer le serveur de développement

Après avoir créé le projet, entrez dans le répertoire du projet et lancez le serveur local de Laravel :

cd monProjetLaravel
php artisan serve

Le serveur local de Laravel sera accessible à l'adresse http://localhost:8000.

Structure d'un projet Laravel

Laravel suit une structure de répertoires très organisée qui facilite la gestion du code. Voici les principaux répertoires d'un projet Laravel :

  • app/ : Contient le code de l'application (contrôleurs, modèles, etc.).
  • resources/ : Contient les vues et les fichiers de ressources comme les CSS et JS.
  • routes/ : Contient les fichiers de définition des routes de l'application.
  • database/ : Contient les migrations, les modèles Eloquent et les usines de données.
  • public/ : Contient les fichiers accessibles publiquement, comme index.php et les assets.

Voici un aperçu de la structure d'un projet Laravel :

monProjetLaravel/
├── app/
│   ├── Http/
│   ├── Models/
│   └── ...
├── resources/
│   ├── views/
│   ├── css/
│   └── js/
├── routes/
│   └── web.php
├── database/
│   ├── migrations/
│   ├── seeds/
│   └── factories/
└── public/
    ├── index.php
    ├── css/
    └── js/

Création de votre première route et contrôleur Laravel

Laravel permet de définir facilement des routes dans le fichier routes/web.php. Voici comment créer une route basique :

Route::get('/', function () {
    return view('welcome');
});

Ceci affiche la vue welcome.blade.php lorsqu'un utilisateur accède à la racine de votre site.

Création d'un contrôleur

Les contrôleurs dans Laravel permettent de centraliser la logique métier. Pour créer un contrôleur, vous pouvez exécuter la commande suivante :

php artisan make:controller HomeController

Cette commande crée un contrôleur HomeController dans le répertoire app/Http/Controllers.

Voici un exemple de méthode dans le contrôleur pour retourner une vue :

class HomeController extends Controller
{
    public function index()
    {
        return view('home');
    }
}

Ensuite, vous pouvez définir une route pour ce contrôleur :

Route::get('/home', [HomeController::class, 'index']);

Exercice :

  • Créez un projet Laravel et définissez une route qui affiche un message personnalisé dans une vue.
  • Créez un contrôleur ProductController avec une méthode pour afficher une liste de produits.
  • Définissez une route qui appelle cette méthode et affiche les produits dans une vue.
